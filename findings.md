# 志愿智选 — 研究发现

## 来源：原报告提取

### 问题定义
- 核心问题：高考志愿填报信息不对称，千万家庭在2-3天内需要做出影响一生的决策
- 目标用户：2026届高考考生及家长（约1300万考生 × 2-3位家长 = 3000-4000万用户）
- 使用场景：高考出分后（6月23-25日各省陆续出分），用户输入省份、选科、分数/位次，系统生成冲稳保推荐方案

### 可行性评估
- 技术可行性：高。核心依赖LLM推理 + 25万条录取数据查询，无需突破性技术
- 数据源风险：25万条数据的准确性、时效性、省份覆盖度是关键
- 时间窗口：每年6月-7月是核心使用窗口

### 竞争分析
- 现有竞品：优志愿、蝶变志愿、高考直通车等志愿卡（200-500元），张雪峰团队咨询服务
- 差异化优势：免费/低价 + 张雪峰方法论AI化 + 透明推荐逻辑
- 进入壁垒：数据源获取、品牌信任建立、用户获取成本

### 风险
- 数据准确性责任（选错志愿后果严重）
- 年度窗口期错过即等一年
- 政策变化风险

## 数据源调研（2026-06-30）

### 发现1：掌上高考网(gaokao.cn)公开JSON API

- 学校列表API：`https://static-data.gaokao.cn/www/2.0/school/school_code.json`
- ✅ **已验证可访问**，返回全国高校ID和名称
- 数据格式：`{"code":"0000","message":"成功","data":{"112002700":{"school_id":"30","name":"北京工业大学"},...}}`
- 录取分数线页面：`https://www.gaokao.cn/school/{school_id}/provinceline`
- 录取数据API：通过抓包分析可得，字段包含：最低分、最高分、平均分、最低位次、录取人数等

### 发现2：付费API（gugudata.com）

- 历年高考高校录取分数线：1999元/年（百万级数据，2015-2025）
- 历年高考专业录取分数线：2499元/年（千万级数据）
- ❌ 太贵，不适合MVP阶段

### 发现3：GitHub学术数据集

- https://github.com/chenowl/Gaokao-Dataset
- 山西省2016-2019年高考数据
- ⚠️ 数据有限，只有1个省且年份较旧

### 发现4：Kaggle

- ❌ 没有找到直接相关的中国高考录取数据集
- 有一些合成数据集，不适用于实际产品

### 数据获取策略决定（已修订 — 2026-06-30）

**❌ 原决定：先用模拟数据跑通产品流程，再用爬虫抓取真实数据替换**

**修订原因（第一性原理复盘）**：

原推理存在严重逻辑错误——混淆了"验证产品流程"与"验证产品价值"：

1. **物理约束误判**：原推理认为"产品流程（表单→算法→PDF）不依赖真实数据就能验证"。
   - 这只对"流程验证"成立，对"产品价值验证"完全不成立。
   - 志愿填报产品的核心价值就是数据准确性，没有真实数据，产品就是0价值甚至是负价值（误导考生）。

2. **风险等级错配**：志愿填报是高 stakes 决策（影响一生），不同于普通电商/工具类MVP。
   - 用模拟数据上线 = 让考生基于假数据做真决策 = 可能造成不可逆的志愿填报错误。
   - 这不是"先简化的五步法"，这是"用错数据自动化"——五步法的反模式。

3. **Idiot Index反推**：竞品（优志愿卡200-500元）的核心壁垒正是"数据准确性"。
   - 如果我们用模拟数据，就完全放弃了这个壁垒，连基本产品形态都不成立。
   - 第一性原理：产品的物理本质 = 真实数据 + 透明算法。算法已就位，数据必须真实。

4. **正确做法**：先删除模拟数据→获取真实数据（哪怕只有1个省的1批学校）→简化范围→上线。

**结论：必须替换为真实数据，无可商量。** 下一步是讨论"如何获取真实数据"。
